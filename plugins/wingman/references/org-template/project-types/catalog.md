# Project-type catalog

A short index for `discovery.md` to consult when a project's shape clearly matches one of these —
never force a fit; most projects are close to one type but not exact, and the discovery
conversation itself always takes precedence over a category label.

| Type | Use when | Playbook |
|---|---|---|
| Website | Marketing/informational site, no user accounts or persistent app state | `website.md` |
| Freelancing delivery | Building for a paying client, not the founder's own product | `freelancing.md` |
| Mini SaaS | Single-feature paid tool, one or two screens, narrow user base | `mini-saas.md` |
| Full SaaS | Multi-feature product, accounts, billing, meant to scale | `full-saas.md` |
| AI agent | The product itself is an agent/assistant, not a traditional app | `ai-agent.md` |
| Automation | Internal script/workflow, no end-user-facing surface | `automation.md` |
| Internal tool | Used by the founder's own team, not sold externally | `internal-tool.md` |

Each playbook is short (what changes about discovery/architecture/ship for this type — not a full
re-derivation of the pipeline). If nothing fits, proceed with the normal pipeline unmodified; forcing
a category is worse than skipping one.
